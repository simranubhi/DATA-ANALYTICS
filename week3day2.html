Repositories in R

A repository in R is a storage location that hosts R packages. Think of it like an app store for R â€” you can search, download, and install packages based on your needs. You can switch between repositories or add your own.

There are 3 main types of repositories:

  • 1. CRAN (Comprehensive R Archive Network): This is the official and most widely used repository. It stores thousands of packages and connects directly to RStudio through secure FTP servers. It's stable, secure, and constantly updated.
  • 2. Bioconductor: A specialized repository mainly used for bioinformatics and computational biology. It includes packages for genomic data analysis and visualization.
  • 3. GitHub: A platform for open-source development. It's used to host R packages not yet available on CRAN. Developers use GitHub to collaborate, share, and improve packages. It supports version control and has unlimited hosting.

Viewing and Managing Repositories

# Shows the current library paths where packages are installed
.libPaths()

# Opens the path location in your system's file explorer
shell.exec(.libPaths()[1])

Viewing Available & Installed Packages

# Lists all available libraries (installed + standard)
library()

# Lists all installed packages in your R setup
installed.packages()

Installing Packages from CRAN

To install any package from CRAN, use the install.packages() function. Example:

# Installs the 'dplyr' package
install.packages("dplyr")

Updating Packages

# Updates the package 'ggplot2' if an update is available
update.packages("ggplot2")

Installing from Bioconductor

First, install the BiocManager package. Then install any Bioconductor package using BiocManager:

install.packages("BiocManager")      # Installs BiocManager
BiocManager::install("edgeR")        # Installs 'edgeR' package from Bioconductor

Installing Packages from GitHub

To install packages from GitHub, use the devtools package:

install.packages("devtools")                     # Installs devtools
devtools::install_github("tidyverse/tidyverse")  # Installs tidyverse from GitHub repo

Loading Packages

You can load one or more packages using library() or require():

library(dplyr)                          # Loads dplyr
require(dplyr)                          # Loads dplyr (returns TRUE/FALSE if successful)
library(caret, dplyr, ggplot2)          # Loads multiple packages at once

Summary

In R, understanding how repositories work is crucial for managing packages. CRAN is your go-to for reliable packages, Bioconductor for bioinformatics, and GitHub for cutting-edge or experimental tools. Always keep your packages updated and organized using functions like library(), install.packages(), and update.packages().
